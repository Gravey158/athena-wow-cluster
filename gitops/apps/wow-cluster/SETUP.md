# wow-cluster — Setup notes

WoW realm namespace. Three logical layers:

1. **Galera** (mariadb-operator): `wow-galera` MariaDB CR with 3 replicas, plus `Database`/`User`/`Grant` CRs.
2. **ToCloud9 services** (Helm chart at `chart/` in this repo): authserver, gateway, gameserver_ac, microservices (chat/mail/guild/...), NATS, Redis.
3. **External LoadBalancer services** (MetalLB-pinned `.72` auth, `.73` gateway) — see `external-services.yaml`.

## Sealed-Secret seeding (one-time, before any of the three layers can sync)

The MariaDB CR + Helm chart together reference **three** Secrets, all sealed by one script:

| Secret | Used by | Generated by |
|---|---|---|
| `mariadb-root` | MariaDB CR (`rootPasswordSecretKeyRef`) | seal-mariadb-creds.sh |
| `mariadb-acore` | User CR `acore` (`passwordSecretKeyRef`) | seal-mariadb-creds.sh |
| `wow-databases-config` | every ToCloud9 service template (`valueFrom: secretKeyRef`) | seal-mariadb-creds.sh |

```bash
cd /Users/dennis/athena-wow-cluster
scripts/seal-mariadb-creds.sh
git add gitops/apps/wow-cluster/{mariadb-creds-sealed-secret.yaml,wow-databases-config-sealed-secret.yaml}
git commit -m "chore(wow-cluster): seal MariaDB + databases-config secrets"
git push
```

The script prints both passwords on stdout — save them for emergency root access.

## Realm-list row (post first deploy, before client login)

ToCloud9 expects the `realmlist` table in `acore_auth` to point at the gateway LB:

```bash
kubectl -n wow-cluster exec -it wow-galera-0 -- \
  mariadb -u acore -p"$ACORE_PWD" acore_auth -e \
  "UPDATE realmlist SET address='10.10.30.73', port=32766 WHERE id=1;"
```

Or seed via initial SQL job in Phase 0.5.

## Client-side (realmlist.wtf)

In `/Users/dennis/athena-cluster/ChromieCraft_3.3.5a/Data/<locale>/realmlist.wtf`:

```
set realmlist 10.10.30.72
```

(Port 3724 is implicit for the auth handshake; gateway port comes from the realmlist row above.)

## Sanity check after Galera + chart sync

```bash
kubectl -n wow-cluster get mariadb wow-galera                            # Ready
kubectl -n wow-cluster get pods                                          # 3 galera + ToCloud9 services
kubectl -n wow-cluster get svc                                           # wow-galera*, wow-tocloud9-*, wow-authserver-lb, wow-gateway-lb
kubectl -n wow-cluster get database,user,grant                           # all Ready

# Smoke test DB connectivity from a ToCloud9 service:
kubectl -n wow-cluster exec deploy/wow-tocloud9-authserver -- \
  sh -c 'echo "SHOW DATABASES;" | mysql -h wow-galera-primary -u acore -p"$(echo $AUTH_DB_CONNECTION | sed -nE "s/.*:([^@]+)@.*/\1/p")"'

# Confirm MetalLB pinned IPs:
kubectl -n wow-cluster get svc wow-authserver-lb wow-gateway-lb
# Expected EXTERNAL-IP: 10.10.30.72 and 10.10.30.73
```
