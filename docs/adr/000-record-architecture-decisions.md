# 000. Record architecture decisions

Date: 2026-05-13

## Status

Accepted

## Context

We need to record the architectural decisions made on this project.

## Decision

We will use Architecture Decision Records, as
[described by Michael Nygard](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions).

## Consequences

- ADRs live in `docs/adr/NNN-title.md`.
- Each ADR is numbered sequentially and immutable once accepted.
- A superseding decision references the original by number and updates its Status to "Superseded by ADR NNN".
- ADRs 000-099 reserved for Phase 0; 100-199 for Phase 1; 200-299 for Phase 2; etc.
